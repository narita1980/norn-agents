## Q&A 想定

### Q1. Azure のコストは？

> デモ規模では Container Apps Consumption + Azure OpenAI gpt-4.1-mini で **月数千円程度** を想定。1 PR 合議は 4 回の LLM 呼び出し（3 女神 + モデレーター）。本番はキャッシュ・トークン上限で制御します。

### Q2. なぜ HITL？ 完全自動の方が Agentic では？

> コードレビューは **心理的安全性** が品質に直結します。若手が「準備できた」と思ったタイミングで開始する設計は、現場の信頼獲得に不可欠です。開始後のパイプラインは完全自動です。

### Q3. Copilot / GitHub Advanced Security との違いは？

> Copilot は汎用アシスタント。Norn は **メンタリング特化** — 3 視点合議、成長ダッシュボード、トーン制御。GitHub 上のフローに **寄り添う** 伴走役です。

### Q4. SSE が in-memory なのは本番向き？

> ハッカソン・デモは workers=1 で十分。Phase 5 で Redis EventBus に差し替え予定。API 設計は変えません。

### Q5. 合議品質はどう担保？

> 固定逐次 + Moderator の **構造化 JSON**（must_fix / next_pr / growth / tone）。Ruff 静的解析をコンテキストに付与。ペルソナごとの Few-shot プロンプトでトーンを固定。

### Q6. スキルレベル別対応は？

> デモではテストユーザー（若手/中級/上級）切替。プロンプト方針ブロックで説明深度を変え、**PR 合議にも反映** 済み。本番は GitHub OAuth + users テーブル予定。

### Q7. 失敗時は？

> セッション status が `failed`、GitHub に失敗通知コメント。Webhook は 202 で即返却、合議は BackgroundTasks — GitHub をブロックしません。
